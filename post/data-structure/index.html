資料結構

想學好資料結構的第一步就要先知道，資料結構本身和演算法的關係密不可分，所以兩本課本內容相似的地方非常多，若要說差在哪裡， 資料結構重視資料的儲存方式，演算法在乎取用這些資料的方式。一個好的資料結構並不一定代表儲存空間越小，因為使用越小的儲存 空間可能會帶來取用的不便利，也就是說會加重演算法的負擔。而根據目的的不同選擇不同的演算法及資料結構，使其可應對大多數狀況 則是工程師的工作。

簡單的排序問題:

//the programe using c++ and vector
#incude<iostream>
using namespace std;

void bubble_sort(vector<int>vec){
    for(int ix=0;ix<vec.size();ix++){
        for(int jx=ix;jx<vec.size();jx++){
            if(vec[jx]>vec[jx+1]){
                swap(vec[jx],vec[jx+1]);
            }
        }
    }
}
